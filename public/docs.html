APIForge Documentation

Everything you need to create, publish, and monetize APIs.

Quick Start (60 seconds)

Get from zero to a live API in three steps:

# 1. Create account
POST /api/v1/auth/register
{ "email": "you@email.com", "username": "you", "password": "Pass1234!" }

# 2. Create API instantly (with your token)
POST /api/v1/quick/tasks
Authorization: Bearer YOUR_TOKEN

# 3. Use your API (with the key you received)
GET /api/v1/live/tasks/tasks
X-API-Key: afk_live_xxxxx
You can also create APIs by chatting with our AI at /chat.html

Authentication

APIForge uses two authentication methods:

MethodHeaderUsed For
JWT TokenAuthorization: Bearer TOKENManaging your account, creating APIs
API KeyX-API-Key: afk_live_xxxConsuming generated APIs

JWT tokens expire after 7 days. API keys don't expire unless you set an expiration or revoke them.

Error Handling

All errors return JSON with an error field:

{ "error": "Not Found", "message": "Route GET /xyz not found" }
CodeMeaningSolution
400Bad RequestCheck your request body/params
401UnauthorizedAdd/fix your token or API key
403ForbiddenCheck permissions or plan limits
404Not FoundCheck endpoint URL or resource ID
429Rate LimitedWait or upgrade your plan
500Server ErrorContact support

Quick Create

Create an API with a single word. Available quick APIs:

POST/api/v1/quick/:nameAuth required

Available names: tasks, contacts, notes, bookmarks, inventory, leads, passwords, expenses

// Response
{
  "apiKey": "afk_live_xxxx...",
  "api": { "name": "Tasks", "slug": "tasks", "baseUrl": "/api/v1/live/tasks" }
}

AI Generator

Describe your API in natural language and get a complete schema:

POST/api/v1/ai/generatePublic
{ "description": "restaurant API with menu items, reservations, and reviews" }

To generate AND deploy in one step:

POST/api/v1/ai/deployAuth required

Chat Creator

Have a conversation with our AI to iteratively build your API:

POST/api/v1/chatAuth required
{ "message": "I need a contacts API with name and email", "sessionId": null }
You can also use the interactive chat at /chat.html

Live API Endpoints

Every generated API gets automatic CRUD endpoints:

GET/api/v1/live/:slug/:resourceList all
POST/api/v1/live/:slug/:resourceCreate
PUT/api/v1/live/:slug/:resource/:idUpdate
DELETE/api/v1/live/:slug/:resource/:idDelete

All list endpoints support: ?page=1&limit=20&sort=name:asc&filter={"status":"active"}

SDKs

Auto-generated client libraries for every API:

GET/api/v1/sdk/:slug/:languagePublic

Languages: javascript, typescript, python, curl

Batch Operations

Execute up to 50 operations in a single request:

POST/api/v1/batch/:slugAPI Key
{ "operations": [
  { "method": "POST", "path": "/tasks", "body": { "title": "Task 1" } },
  { "method": "POST", "path": "/tasks", "body": { "title": "Task 2" } },
  { "method": "GET", "path": "/tasks" }
]}

Plugins

Extend your API with pre-built plugins:

GET/api/v1/plugins/availableList plugins
POST/api/v1/plugins/:slugAdd plugin

Available: rate-limiter, cache, transform, cors-custom, ip-whitelist, request-validator, analytics-enhanced, webhook-on-change

Debug & Fix

Diagnose and fix issues with your APIs automatically:

GET/api/v1/debug/:slugFull diagnostic
POST/api/v1/debug/:slug/testAuto-test CRUD
GET/api/v1/debug/:slug/fixAuto-fix suggestions
